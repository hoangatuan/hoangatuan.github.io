## Customise Xcode build phases

It‚Äôs easy to run some scripts in the pipeline if you can control the CI configuration file:

```bash

  - build

  stage: build  
  script:
```

Since we don‚Äôt have permission to customize the CI configuration file, we need to find another way to run the script on CI.

[Xcode build phases](https://developer.apple.com/documentation/xcode/customizing-the-build-phases-of-a-target) can help us with that.

Currently, whenever we create/update a Pull request, a unit test job will run (Yes, it is set up by the infra team). With Xcode build phases, we can inject custom scripts during build time, and CI runner will help us to run it üòõ

Good, it works. This approach can solve some of the simple use cases. However, it has some downsides:

- My laptop will do a lot of heavy work: Pull the heavy project, run the tests, do custom script, ‚Ä¶ which will impact my daily work.
- Can‚Äôt utilize pre-defined jobs provided by the infra team: Gather test coverage of specified modules, performance analysis, ‚Ä¶

<img src="/src/assets/content/post/how-to-customize-ios-ci-workflows-when-you-dont-control-the-ci-system/1_YW3GkI7sSS9KZOIFFZtu6w.png" alt="image" style={{ width: '100%', height: 'auto' }} />

To overcome these downsides, my simple idea is:

- Develop a Python script to trigger the pre-defined jobs by cURL (with the help of the [Python-Jenkins](https://python-jenkins.readthedocs.io/en/latest/) package).
- Proceed the job‚Äôs log to get the information that we want
- Send the final result to the work channel using [webhook](https://www.techtarget.com/searchapparchitecture/tip/Webhooks-explained-simply-and-how-they-differ-from-an-API).
